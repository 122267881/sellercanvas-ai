# SellerCanvas AI 中文开发总览

本文档给项目负责人阅读，用中文说明当前架构、已经完成的内容、后续开发顺序和验收标准。

## 产品目标

SellerCanvas AI 要做成可商用的跨境电商 AI 商品图与 Listing 交付平台，而不是展示 Demo。

核心闭环是：

1. 客户注册登录。
2. 客户购买订阅套餐，获得积分。
3. 客户上传商品图。
4. 系统分析图片，识别商品特征、卖点、材质、平台风险和优化方向。
5. 系统根据分析结果反推出高质量 Prompt。
6. 系统调用生图能力生成商品图。
7. 系统同步生成 Listing 标题、五点描述和详情文案。
8. 客户导出 Listing Pack。
9. 后台记录项目、积分、支付、任务和导出历史。

## 两个网站必须分开

客户网站只给买家使用：

- 注册、登录、购买套餐
- 查看积分
- 创建项目
- 上传图片
- AI 分析
- 生成图片
- 生成文案
- 导出结果
- 查看自己的历史记录

开发者管理后台只给你和运营人员使用：

- 客户管理
- 订阅和支付记录
- 积分流水
- AI Provider 配置
- 开发者 API Key 管理
- API 调用统计
- 审计日志
- 数据导出

客户页面绝对不能出现开发者 API Key、Provider 密钥、全局客户列表等内部功能。

## 当前已经完成的迁移基础

已经建立新架构目录：

- `apps/customer`：未来客户网站
- `apps/admin`：未来开发者后台
- `apps/api`：未来生产后端
- `workers/ai`：未来 AI 任务执行器
- `packages/shared`：平台规则和通用配置
- `packages/billing`：积分账户和积分流水
- `packages/ai-core`：Prompt 底层逻辑
- `prisma`：生产数据库结构和迁移文件
- `legacy`：旧版原型保留说明

已经完成的新后端能力：

- `/api/v2/credits/balance` 查询客户积分
- `/api/v2/jobs` 创建 AI 任务并冻结积分
- `/api/v2/jobs/:id/succeed` 成功后扣除冻结积分
- `/api/v2/jobs/:id/fail` 失败后退回冻结积分
- Stripe Webhook 服务模块
- Prisma 持久化适配器
- API 权限分层

已经完成的旧站桥接：

- 注册后赠送试用积分
- 订阅确认后发放套餐积分
- 商品分析扣积分
- 批量生图扣积分
- 重写 Listing 文案扣积分
- 导出 Listing Pack 扣积分
- 手动保存文案不扣积分
- 客户页面显示当前可用积分

## 积分扣费规则

当前基础价格：

- 商品图分析：5 积分
- Prompt 生成：3 积分
- 4 张商品图批量生成：80 积分
- Listing 文案生成：3 积分
- 导出打包：2 积分

真实商业环境建议：

- 后续可按模型成本、图片尺寸、生成张数、平台模板复杂度继续拆分价格。
- 所有扣费必须由后端执行，不能相信前端传来的价格。
- 失败任务必须退回冻结积分。

## 接下来开发顺序

1. 把旧客户站继续瘦身，只保留客户真正需要的购买和使用流程。
2. 把管理后台继续独立出来，只保留开发者和运营功能。
3. 把内存版积分仓库替换为 Prisma 数据库存储。
4. 把 AI Worker 做成独立进程，支持重试、失败退回和任务日志。
5. 接入真实 OpenAI 图片分析、Prompt 生成、Listing 文案和生图 API。
6. 补齐 Stripe 正式订阅、Customer Portal、发票和 Webhook 幂等处理。
7. 增加生产部署脚本、环境变量检查和上线文档。

## 验收标准

商用版本至少要满足：

- 客户能独立注册、购买、上传、生成、导出。
- 客户所有付费功能都按积分扣费。
- 积分不足时不能继续生成，并给出明确提示。
- 开发者后台和客户网站严格隔离。
- API Key 和 Provider 配置只在后台出现。
- 支付、积分、任务、导出都有历史记录。
- Docker 部署后可以直接运行。
- README 和部署说明全部提供中文版本。

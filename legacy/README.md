# 旧版原型保留规则

根目录现有原型在新架构迁移完成前必须保持可运行。

旧版根目录文件包括：

- `index.html`
- `admin.html`
- `app.js`
- `admin.js`
- `server.js`
- `styles.css`
- `data/db.json`
- `exports/`

这些文件现在的定位：

- 作为当前可运行版本，保证你随时能打开和测试
- 作为业务行为参考，帮助新架构迁移
- 逐步桥接到新的积分、任务、支付和后台模块

不要提前删除或移动这些文件。只有当新客户站、新管理后台、新 API 和 Worker 全部通过 MVP 验收后，才开始清理旧代码。
